# 📖 Overview

This repository contains the deliverables for **Week 1** of my Cybersecurity Internship.

The objective of this phase was to set up a vulnerable web application, perform an initial security assessment, identify potential vulnerabilities, and document the findings using both automated and manual testing techniques.

The assessment established the security baseline for implementing improvements in the following weeks.

---

# 🎯 Objectives

- 💻 Set up the web application locally
- 🔍 Perform vulnerability assessment
- 🛡️ Identify common web security issues
- 📋 Document security findings
- 📊 Establish a baseline for security improvements

---

# 🚀 Activities Performed

### 💻 Application Setup

The web application was successfully configured in a local development environment.

Tasks completed:

- Installed project dependencies
- Configured the application
- Launched the server on **localhost**
- Explored application features including:
  - User Registration
  - User Login
  - Dashboard
  - Profile

---

### 🔍 Vulnerability Assessment

Security testing was performed using both automated and manual techniques.

Assessment included:

- Automated vulnerability scanning with **OWASP ZAP**
- Manual browser-based security testing
- Input validation testing
- Authentication testing
- Session management testing
- Basic SQL Injection testing
- Cross-Site Scripting (XSS) testing

---

### 🛡️ Vulnerabilities Identified

The assessment identified several security concerns, including:

- Missing HTTP Security Headers
- Content Security Policy (CSP) configuration issues
- Cross-Origin Resource Sharing (CORS) recommendations
- Weak authentication controls
- Lack of account lockout after multiple failed login attempts
- Session management observations

These findings formed the basis for implementing security improvements in Week 2.
